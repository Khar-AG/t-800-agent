# Hooks — реакции Cursor на события

## Простыми словами

Hooks — это правила «когда случилось X, выполни Y». Например, при старте сессии очистить handoff-файл или запустить проверку.

## Когда это нужно

- У команды есть строгий pipeline
- Нужно автоматически готовить окружение
- Нужно проверять файлы перед задачей

## Почему это продвинутый уровень

Hook может запускать команды. Неправильный hook может мешать работе или портить файлы. Новичкам лучше сначала использовать rules и skills.

## Мини-чеклист безопасности

- Команда понятна
- Нет удаления без backup
- Нет секретов в файлах
- Hook нужен всем в проекте, а не только одному человеку
