# Типичные проблемы новичков Cursor

## Не вижу Agent

**Суть:** Agent может быть скрыт, открыт не тот режим или не та панель.

Что сделать:

1. Откройте чат/Agent через интерфейс Cursor
2. Проверьте режим: Ask, Plan, Agent или Debug
3. Откройте папку проекта, а не один случайный файл

## Не понимаю diff

**Суть:** зелёное — добавлено, красное — удалено.

Представьте редактора, который показывает правки в документе до принятия.

Что сделать:

1. Не принимайте diff сразу
2. Попросите Agent объяснить каждую группу изменений
3. Принимайте только понятное

## Агент всё сломал

**Суть:** сначала смотрим diff/checkpoint, потом исправляем.

Что сделать:

1. Не запускайте новые большие задачи
2. Найдите checkpoint перед правкой
3. Попросите объяснить, какие файлы были изменены
4. Откатите или попросите минимальный фикс

## Боюсь терминала

**Суть:** терминал — это пульт проекта, но новичку не нужно нажимать всё подряд.

Что сделать:

1. Просите Agent сначала объяснить команду
2. Не соглашайтесь на удаление файлов без понимания
3. Для опасных команд спрашивайте: «что изменится и как откатить?»

## MCP не появился

**Суть:** часто нужен перезапуск Cursor, правильный config и секреты.

Что сделать:

1. Проверьте, куда добавлен MCP config
2. Проверьте API keys/env
3. Перезапустите Cursor
4. Спросите Agent: «какие MCP tools доступны?»

## Нет Publish/Share в Canvas

**Суть:** кнопка может быть не у файлового `.canvas.tsx`, а у Canvas, созданного/открытого в интерфейсе; ещё влияют тариф, team и Privacy Mode.

Что сделать:

1. Откройте Canvas как canvas в Cursor
2. Ищите `Publish`, не обычный `Share`
3. Проверьте team/тариф
4. Проверьте, не включён ли Legacy Privacy Mode

## Agent просит слишком много прав

**Суть:** это момент остановиться и перейти в Plan/Ask.

Что сделать:

1. Попросите объяснить риск
2. Попросите разбить задачу на маленькие шаги
3. Разрешайте только понятные действия
