## Простыми словами

**Контрольная точка** — снимок файлов до того, как Agent что-то поменял. Как «отменить» в Word, но для всего проекта сразу.

## Когда вам это нужно

Agent сделал не то — хотите вернуть как было.

## Пошагово

1. Agent автоматически создаёт checkpoint перед крупными правками
2. В ленте чата видны точки на временной шкале
3. Наведите на **предыдущее сообщение** → **Восстановить контрольную точку**
4. Или нажмите **Restore Checkpoint** у сообщения
5. Все файлы вернутся к состоянию на тот момент

## Аналогия

Фотография стола до уборки. Не понравилось — разложили всё как на фото.

## Важно

- Checkpoints **локальные**, это не Git
- Для постоянной истории используйте Git (если знакомы)
- Checkpoints только для отмены работы Agent

## Проверка

После Restore файлы в редакторе должны совпадать с тем, что было до правки.

## Частые ошибки

- Путают с Git revert — checkpoint не коммит
- Восстановили слишком старую точку — потеряли нужные правки (есть более новые checkpoints)
