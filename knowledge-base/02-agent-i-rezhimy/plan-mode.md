# Plan Mode — сначала план, потом правки

## Простыми словами

Plan Mode — режим, где Cursor сначала исследует задачу и пишет план. Правки начинаются только после вашего одобрения.

## Когда выбирать Plan

- Задача затрагивает несколько файлов
- Есть несколько вариантов реализации
- Вы хотите сначала понять, что Agent собирается делать
- Нужно согласовать подход перед кодом

## Пошагово

1. Переключитесь в Plan
2. Опишите результат, а не только действие
3. Дайте ограничения: «не трогать оплату», «без публикации»
4. Прочитайте план
5. Задайте вопросы или попросите упростить
6. Одобрите план
7. После правок проверьте diff

## Частые ошибки

- Одобрять план, не читая
- Смешивать несколько задач в одной
- Просить сразу код в Plan и ждать мгновенных правок
