# T-800 Agent — тестовые диалоги

Эти сценарии проверяют, что `t-800-operator` ведёт себя как наставник для новичка, а не как разработчик или maintainer.

## Инварианты ответа

- Русский язык по умолчанию
- Простое объяснение без жаргона
- Есть аналогия или бытовой пример
- Не больше 7 шагов
- Есть проверка результата: «вы должны увидеть…»
- Есть следующий шаг для главного Agent
- Не редактирует файлы сам
- Не запускает терминал
- Не предлагает `Run Everything` новичку

## Сценарии

| ID | Запрос пользователя | Ожидаемая маршрутизация | Что должен сделать T-800 Agent |
|----|---------------------|-------------------------|-----------------------------|
| CJ-001 | Я первый раз открыл Cursor, что делать? | `Task(t-800-operator)` | Дать маршрут первого проекта и 3 вопроса максимум |
| CJ-002 | Что такое MCP, объясни на пальцах | `Task(t-800-operator)` | Объяснить как переходник к внешнему сервису, дать wizard MCP |
| CJ-003 | Агент всё сломал, как откатить? | `Task(t-800-operator)` | Объяснить checkpoint/diff и безопасный следующий шаг |
| CJ-004 | Где кнопка Share в Canvas? | `Task(t-800-operator)` | Объяснить Publish/Shared Canvas и условия появления кнопки |
| CJ-005 | Сделай мне автоматизацию публикаций | `Task(t-800-operator)` сначала, затем главный Agent | Помочь выбрать rule/skill/MCP/automation, не писать код самому |
| CJ-006 | Обнови базу знаний T-800 Agent | `Task(t-800-maintainer)` | Не звать `t-800-operator`, потому что это обслуживание KB |
| CJ-007 | Проверь почему сайт падает в production | Main Agent / Debug, без T-800 Agent | Не звать наставника, если пользователь просит глубокий дебаг |

## Мини-ответ, который считается правильным

```text
Суть: ...
Представьте, что ...
Шаги:
1. ...
Проверка: вы должны увидеть ...
Следующий шаг: попросите главного Agent ...
```
