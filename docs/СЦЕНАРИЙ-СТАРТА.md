# Сценарий старта T-800 — 4 шага

**Версия плагина:** 1.10.0  
**Аудитория:** новички, первый раз ставят T-800  
**Способ:** без ручного терминала — Cursor делает установку по промпту

Это **канонический сценарий** онбординга. Дублируется в:

- `share/T-800-ИНСТРУКЦИЯ.md` — для рассылки команде
- `docs/НАЧАЛО-РАБОТЫ.md` — краткая инструкция внутри плагина

---

## Обзор

| Шаг | Действие | Кто делает |
|-----|----------|------------|
| **1** | Создать папку проекта | Человек |
| **2** | Положить архив плагина | Человек |
| **3** | Открыть в Cursor + промпт на установку | Cursor Agent |
| **4** | Reload Window + `/t800-bootstrap` | Человек + плагин |

После шага 4 T-800 готов к работе в **любом** проекте Cursor.

---

## Шаг 1 — папка проекта

Создайте пустую папку, например:

```text
t800 agents
```

Можно любое имя. Главное — отдельная папка, которую откроете в Cursor как workspace.

---

## Шаг 2 — архив

Положите в папку файл из рассылки:

```text
t800 agents/
  └── t-800-agent-v1.10.0.zip
```

Опционально — `T-800-ИНСТРУКЦИЯ.md` (эта инструкция рядом с zip).

---

## Шаг 3 — установка через Cursor

1. **File → Open Folder** → выберите `t800 agents`
2. Откройте чат **Agent** (режим Agent, не Ask)
3. Вставьте промпт целиком:

### Промпт установки

```text
В этой папке лежит архив t-800-agent-v1.10.0.zip — плагин T-800 Agent для Cursor.

Сделай всё сам:
1. Разархивируй zip в эту папку (должна появиться папка t-800-agent).
2. Установи плагин: запусти bash scripts/install-plugin.sh из папки t-800-agent, затем bash scripts/verify-install.sh — должно быть verification passed.
3. Напиши мне, что установка прошла успешно, и попроси сделать Reload Window (Developer: Reload Window).
4. После перезагрузки я напишу тебе снова — тогда запусти /t800-bootstrap.
```

### Ожидаемый результат

- Папка `t-800-agent/` с исходниками плагина
- В терминале: `T-800 Agent verification passed`
- Сообщение от агента: сделайте **Reload Window**

> **Windows:** в промпте можно заменить `bash` на `pwsh` и скрипты `.ps1`, если агент на Windows.

---

## Шаг 4 — первый запуск плагина

1. `Cmd+Shift+P` (или `Ctrl+Shift+P`) → **Developer: Reload Window**
2. Снова откройте чат **Agent**
3. Напишите:

```text
/t800-bootstrap
```

Или развёрнуто:

```text
Запусти первый старт T-800: проанализируй мою систему Cursor, расскажи что умеет плагин, и предложи установить глобальное правило T-800. Я новичок.
```

### Что произойдёт

| Этап | Действие плагина |
|------|------------------|
| Аудит | Сканирует `~/.cursor/` и локальный `.cursor/` |
| Обзор | Рассказывает команды и цепочку агентов |
| Правило | Предлагает глобальное routing rule — **рекомендуем «Да»** |

**Готово.** Дальше в любом проекте: `/t800-start` + описание задачи.

## После сценария — ежедневные команды

| Команда | Когда |
|---------|--------|
| `/t800-bootstrap` | Пропустили шаг 4 или нужно повторить первый старт |
| `/t800-onboard` | Посмотреть, что настроено в Cursor |
| `/t800-start` | Создать subagent, skill, rule, command, hook |
| `/t-800-operator` | Вопросы про Cursor |
| `/t-800-health` | Диагностика |

## Если что-то пошло не так

**Агент не распаковал zip** — попросите: «Разархивируй t-800-agent-v1.10.0.zip в текущую папку».

**verify-install failed** — попросите повторить `bash scripts/install-plugin.sh` и показать вывод.

**Команда `/t800-bootstrap` не находится** — не сделали Reload Window после шага 3.

**Ручная установка** (запасной путь):

```bash
cd t-800-agent
bash scripts/install-plugin.sh
bash scripts/verify-install.sh
```

→ Reload Window → `/t800-bootstrap`
