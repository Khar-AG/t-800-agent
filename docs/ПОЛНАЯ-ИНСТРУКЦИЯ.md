## 1. Что это такое

**T-800 Agent** — плагин для Cursor: цех, который помогает **правильно** создавать и настраивать:

| Артефакт | Зачем |
|----------|--------|
| **Субагенты** (`agents/*.md`) | Специализированные помощники в чате |
| **Skills** | Навыки с чеклистами и скриптами |
| **Commands** | Команды вида `/...` |
| **Rules** | Правила поведения агента (в т.ч. global) |
| **Hooks** | Хуки IDE (session, tools, shell…) |
| **Scripts** | install / validate / gate |

Вместо ручной сборки «на глаз» плагин ведёт конвейер:

```text
уточнения → разведка Cursor → автономный поиск в интернете
  → промпт-крафт → база знаний → фабрика → проверка качества
```

Отдел **сам** решает, где искать (GitHub, Reddit, ClawHub, Context7, cookbooks GPT/Claude/Gemini/Perplexity…) и **сам** выбирает лучший вариант из нескольких источников.

## 2. Установка с нуля

Нужны: Cursor, чат в режиме **Agent** (не Ask), архив `t-800-agent-v1.13.0.zip`.

### Шаг 1 — папка

Создайте папку, например:

```text
t800 agents/
```

### Шаг 2 — архив

Положите внутрь:

```text
t800 agents/
  └── t-800-agent-v1.13.0.zip
```

(рядом можно положить эту инструкцию)

### Шаг 3 — открыть в Cursor

**File → Open Folder** → выберите `t800 agents`.

### Шаг 4 — промпт установки

Откройте чат **Agent** и вставьте целиком:

```text
В этой папке лежит архив t-800-agent-v1.13.0.zip — плагин T-800 Agent для Cursor.

Сделай всё сам:
1. Разархивируй zip в эту папку (должна появиться папка t-800-agent).
2. Установи плагин: запусти bash scripts/install-plugin.sh из папки t-800-agent, затем bash scripts/verify-install.sh — должно быть verification passed.
3. Напиши мне, что установка прошла успешно, и попроси сделать Reload Window (Developer: Reload Window).
4. После перезагрузки я напишу тебе снова — тогда запусти /t800-bootstrap.
```

### Шаг 5 — Reload Window

`Cmd+Shift+P` (или `Ctrl+Shift+P`) → **Developer: Reload Window**.

Без Reload новые команды и субагенты в чате не подхватятся.

### Запасной путь (терминал)

```bash
cd путь/к/t-800-agent
bash scripts/install-plugin.sh
bash scripts/verify-install.sh
```

Windows: `pwsh scripts/install-plugin.ps1` и `verify-install.ps1`.

## 3. Обновление (если уже стоит старая версия)

Если раньше ставили 1.5–1.9 и получили новый zip — **не нужно** удалять Cursor. Достаточно переустановить плагин поверх.

### Промпт обновления (скопировать в Agent)

Положите новый `t-800-agent-v1.13.0.zip` в ту же (или любую) папку проекта и напишите:

```text
Обнови плагин T-800 Agent до актуальной версии из этой папки.

Сделай сам:
1. Если есть zip t-800-agent-v*.zip — разархивируй с обновлением папки t-800-agent.
2. bash t-800-agent/scripts/install-plugin.sh
3. bash t-800-agent/scripts/verify-install.sh — нужно verification passed.
4. Покажи version из plugin.json и сколько файлов t-800-*.md в
   ~/.cursor/plugins/local/t-800-agent/agents/
5. Попроси Reload Window.
6. После Reload я смогу использовать /t800-audit и /t800-start.
```

### После установки новой версии

В чате можно писать:

```text
/t800-update
```

(команда покажет ту же процедуру)

### Проверка, что обновилось

Ожидаемо после Reload:

- версия плагина **1.13.0**
- агентов **36** в `~/.cursor/plugins/local/t-800-agent/agents/t-800-*.md`
- команды `/t800-fix`, `/t800-doctor`, `/t800-audit`, `/t800-plugin-audit`, `/t800-start` находятся

## 4. Первый запуск после Reload

В любом проекте (или в папке `t800 agents`) напишите:

```text
/t800-bootstrap
```

Или человеческим языком:

```text
Запусти первый старт T-800: проанализируй мою систему Cursor, расскажи что умеет плагин, и предложи установить глобальное правило T-800. Я новичок.
```

Плагин:

1. просканирует global/local настройки;
2. расскажет возможности;
3. предложит **глобальное правило**: создавать rules/skills/commands/hooks/subagents только через T-800 — рекомендуем ответить **«Да»**.

После этого система готова к работе в **любом** проекте Cursor.

## 6. Что умеет система

### 6.0. Оркестрация отделов (без лишних агентов)

Четыре отдела — **System**, **Research**, **Brains**, **Factory**.  
Новых агентов в Research/Brains не добавляем: лиды сами вызывают нужных специалистов.

| Кто | Что делает |
|-----|------------|
| **Директор** (`/t800-start`) | Зовёт только лидов + показывает progress |
| **research-lead** | Авто: strategist → поиск → synthesizer |
| **brain-lead** | Авто: 1–2 domain brain по теме |
| **factory** | Авто: architect → сборка → QA |

Progress в чате, например: `T-800 ▸ [■■■□□] 3/5 Research готов → дальше Brain…`  
Контракт: внутри плагина `shared/department-orchestration-contract.md`.

### 6.0b. Loop engineering (STATE + machine gates) — v1.12+

Без новых агентов. Минимальный цикл вокруг существующих команд:

| Элемент | Смысл |
|---------|--------|
| **STATE.md** | `{memory_path}/STATE.md` — Last run, In progress, Gates, Lessons |
| **Machine gate** | Канон: `python3 scripts/t800_run_gate.py` + auditor PASS |
| **Repair budget** | После FAIL — до **2** циклов fix→re-audit, затем вопрос человеку |
| **Research mode test** | DEEP / LIGHT / SKIP по сложности задачи, не только по фразе |

Скрипт STATE: `bash scripts/t800_loop_state.sh init|touch --memory-path …`  
Контракт: `shared/loop-engineering-contract.md`.

### 6.0c. Fix / Doctor — v1.13

| Команда | Смысл |
|---------|--------|
| **`/t800-fix`** | audit → `{memory}/fix-packs/<slug>.md` → PATCH (не полный DEEP) |
| **`/t800-doctor`** | быстрый health без narrative-агента |
| **`t800_audit_to_fixpack.py`** | черновик fix-pack из plugin-audit |

Контракт: `shared/fix-pipeline-contract.md`.

### 6.1. Создание «с умом»

По запросу вроде «сделай skill для PDF» или «субагент для code review» отдел:

- уточнит у новичка модели / MCP / куда писать (если неясно);
- сверится с официальными доками Cursor;
- **сам** выберет, где искать дальше;
- сравнит источники и предложит оптимальный вариант;
- соберёт файлы и проверит качество промпта.

### 6.2. Автономный поиск (DEEP)

По умолчанию при `/t800-start` (если не сказали «только KB»):

| Мозг / специалист | Роль |
|-------------------|------|
| **Strategist** | Сам решает *куда* искать |
| GitHub + **repo-miner** | Поиск репо и глубокий разбор (skills, hooks, scripts) |
| Community | Reddit, Habr, HN, X |
| **ClawHub** | Маркетплейс skills: https://clawhub.ai/ |
| **Vendor docs** | GPT Cookbook, Claude, Gemini, Perplexity, Kie.AI, Cursor |
| **Context7** | Живые доки библиотек/API/SDK (не always-on) |
| News | Changelog, свежие блоги |
| **Synthesizer** | Сравнивает всё → один лучший подход |

Вам **не нужно** писать «пойди на GitHub / Reddit» — достаточно цели.

### 6.3. Промпт-инженерия

- Гайды под GPT / Claude / Gemini / Perplexity / Cursor Agent  
- Идеи из OpenAI Cookbook и аналогов  
- QA промптов перед «релизом» артефакта (anti Description Trap)

### 6.4. Разбор вашей системы Cursor

См. [§8](#8-разбор-своей-системы-cursor) — `/t800-audit`.

### 6.5. Помощь по Cursor в целом

`/t-800-operator` — режимы Agent/Ask/Plan, MCP, skills, rules «на пальцах».

## 7. Как создавать артефакты

### Базовый шаблон

```text
/t800-start

<что нужно создать>
Куда: для этого проекта | глобально | для плагина <имя>
Ограничения: например readonly / без интернета / только KB
```

### Примеры

**Субагент**

```text
/t800-start

Нужен субагент для code review React-проекта.
Только анализ, без правок файлов (readonly).
Для этого workspace.
```

**Skill**

```text
/t800-start

Skill для генерации картинок через Kie.AI.
Я новичок — уточни модели и MCP. Для этого проекта.
```

**Rule**

```text
/t800-start

User rule: всегда отвечать на русском. Глобально.
```

**Command**

```text
/t800-start

Команда /my-deploy для чеклиста деплоя на Railway. Для этого проекта.
```

### Цепочка под капотом

```text
intake-clarifier?
  → scout (cursor.com)
  → research-lead DEEP
       strategist → специалисты → synthesizer
  → prompt-craft?
  → brain-lead
  → factory (architect → builder → integrator → prompt-auditor → auditor)
```

## 8. Разбор своей системы Cursor

Многие ставят десятки rules с `alwaysApply: true` — они попадают **в каждый** чат и съедают контекст.

### Запуск

```text
/t800-audit
```

### Что произойдёт

1. Инвентаризация: rules, skills, commands, agents (global + local)  
2. Оценка «жира»: размер + alwaysApply → риск high / medium / low  
3. **Диалог с вами** по кандидатам:
   - что это такое;
   - зачем может быть нужно;
   - **оставить / сузить / удалить / объяснить подробнее**  
4. Итоговая таблица рекомендаций  

Ничего не удаляется без вашего явного «да».

Быстрый обзор без глубокого разбора:

```text
/t800-onboard
```

### Аудит одного плагина (`/t800-plugin-audit`)

Когда нужна **карта** конкретного плагина (Teya, свой fork), а не разбор всей системы Cursor:

```text
/t800-plugin-audit
```

Под капотом: `t800_plugin_audit.py` → `Task(t-800-plugin-auditor)`.  
Отчёт пишется в `{memory_path}/audits/<run-id>/` целевого проекта (не в knowledge-base T-800).

| | `/t800-plugin-audit` | `/t800-audit` |
|--|----------------------|---------------|
| Объект | Один `--plugin-root` | Global + local Cursor |
| Выход | inventory / graph / orphans | диалог keep/narrow/remove |

## 9. Куда пишутся файлы

При `/t800-start` укажите поверхность:

| Вы сказали | Куда пишем |
|------------|------------|
| «для этого проекта» | `проект/.cursor/` |
| «глобально» | `~/.cursor/` (user rules/skills/commands — по согласию / отдельный surface) |
| «для плагина X» | git-checkout плагина (например Teya) |

Плагин T-800 сам живёт в:

```text
~/.cursor/plugins/local/t-800-agent/
```

Субагенты, skills, rules и commands T-800 ставятся **только** в этот каталог плагина (без зеркал в `~/.cursor/agents` и т.п.). Глобальное правило маршрутизации — через `/t800-bootstrap` (consent).

## 11. Частые вопросы

**Команда `/t800-…` не находится**  
→ Не сделали Reload Window после install, или стоит старая версия → [§3 Обновление](#3-обновление-если-уже-стоит-старая-версия).

**Нужно ли каждый раз открывать папку `t800 agents`?**  
→ Нет. После install плагин работает **во всех** проектах. Папка нужна для установки/обновления из zip.

**Можно ли без интернета?**  
→ Да: в задаче напишите «только KB / без интернета» — DEEP research пропустится.

**Удалит ли audit мои rules сам?**  
→ Нет. Только рекомендации и вопросы. Удаление — после вашего согласия.

**Глобальное правило обязательно?**  
→ Нет, но рекомендуется: тогда агент не будет лепить skills/rules руками в обход конвейера.

**Windows**  
→ В промптах можно заменить `bash` на `pwsh` и скрипты `.ps1`.

**Что-то сломалось**  
→ `/t-800-health` или в терминале: `bash scripts/health-check.sh` из папки `t-800-agent`.
