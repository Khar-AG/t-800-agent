## 1. Что это такое

**T-800 Agent** — плагин для Cursor: цех, который помогает **правильно** создавать и настраивать:

| Артефакт | Зачем |
|----------|--------|
| **Субагенты** (`agents/*.md`) | Специализированные помощники |
| **Skills** | Навыки с чеклистами |
| **Commands** | Slash-команды `/...` |
| **Rules** | Правила агента (в т.ч. global) |
| **Hooks** | Хуки IDE |
| **Scripts** | install / validate / gate |

Вместо ручной сборки «на глаз»:

```text
уточнения → разведка → research → brain → factory → machine gate
```

## 2. Установка с нуля

Нужны: Cursor, чат **Agent**, интернет.

### Рекомендуемый путь — GitHub

```bash
git clone https://github.com/Khar-AG/t-800-agent.git
cd t-800-agent
bash scripts/install-plugin.sh
bash scripts/verify-install.sh
```

Ожидайте: `verification passed`. Затем **Developer: Reload Window**.

### Промпт в чате (папка с клоном или zip)

```text
Установи плагин T-800 Agent из этой папки.

Сделай сам:
1. Если есть zip t-800-agent-v*.zip — разархивируй в t-800-agent.
2. bash t-800-agent/scripts/install-plugin.sh
3. bash t-800-agent/scripts/verify-install.sh — нужно verification passed.
4. Покажи version из ~/.cursor/plugins/local/t-800-agent/.cursor-plugin/plugin.json
5. Попроси Developer: Reload Window.
6. После Reload я запущу /t800-bootstrap
```

Канон установки:

```text
~/.cursor/plugins/local/t-800-agent
```

## 3. Обновление

### Автомат (по умолчанию)

**Вручную обновлять каждый раз не нужно.**

При **новом чате** hook `sessionStart`:

1. сравнивает локальную версию с GitHub (API);
2. если remote новее — скачивает архив и ставит;
3. агенту: сообщить пользователю → **Reload Window** → продолжить задачу.

Отключить:

```bash
export T800_SKIP_AUTO_UPDATE=1
```

### Вручную (срочно)

```text
/t800-update
```

```bash
bash ~/.cursor/plugins/local/t-800-agent/scripts/t800-update-from-github.sh
```

Zip в чат для обновления **не обязателен** (остался только как запасной канал для офлайна).

## 4. Первый запуск

После Reload:

```text
/t800-bootstrap
```

1. Обзор global + local (rules, skills, commands, agents, plugins).
2. Предложение глобального правила: артефакты Cursor — только через T-800.
3. Подсказка: `/t800-start`, `/t800-audit`, `/t800-doctor`.

Повторный обзор без «первого раза»: `/t800-onboard`.

## 5. Все команды

### Ежедневные

| Команда | Назначение |
|---------|------------|
| `/t800-start` | Создать skill / agent / command / rule / hook |
| `/t800-fix` | Точечная правка по fix-pack |
| `/t800-audit` | Аудит системы Cursor (лишнее, alwaysApply) |
| `/t800-plugin-audit` | Аудит одного плагина |
| `/t800-doctor` | Быстрый health |
| `/t800-bootstrap` | Первый запуск |
| `/t800-onboard` | Обзор без глубокого аудита |
| `/t800-cloud-hub` | Blank Hub + Client Cursor Automations |
| `/t800-update` | Ручное обновление с GitHub |

### Служебные

| Команда | Назначение |
|---------|------------|
| `/t800-hub-setup` | Алиас `/t800-cloud-hub` |
| `/t-800` | Алиас `/t800-start` |
| `/t-800-factory` | Только factory |
| `/t-800-factory-validate` | Validate артефактов |
| `/t-800-operator` | Обучение Cursor для новичка |
| `/t-800-maintain` | Память / docs отдела |
| `/t-800-sync` | Синхронизация |
| `/t-800-health` | Техдиагностика |

### Как работает `/t800-start`

```text
discovery → intake? → scout → research-lead → brain-lead → factory → auditor
```

Surface: плагин / проект (`.cursor`) / user (`~/.cursor`).  
Progress между отделами + `STATE.md` в memory проекта.

### Как работает `/t800-fix`

Fix-pack → factory **PATCH** (только `files[]`) → gate.  
Research: SKIP/LIGHT по умолчанию.

### Как работает `/t800-audit`

Machine scripts + `Task(t-800-system-auditor)` → диалог keep/narrow/remove → при необходимости fix-pack → `/t800-fix`.

### Как работает `/t800-plugin-audit`

`t800_plugin_audit.py` → narrative auditor → `t800_audit_to_fixpack.py` → `/t800-fix`.

### Как работает `/t800-cloud-hub`

```text
Task(t-800-cloud-hub-lead) → analyst | prompt | pack | smoke → {memory}/cloud-hub/
```

Отдел **Cloud Hub**: blank Hub (умения = checkout плагина) + Client TZ-builder.  
Контракт: `shared/cloud-hub-setup-contract.md`. Алиас: `/t800-hub-setup`.

## 6. Примеры промптов

### Проверить свой Cursor

```text
/t800-audit

Проверь мой Cursor: глобальные и локальные rules, skills, commands, субагенты.
Найди alwaysApply, дубли и то, чем я не пользуюсь.
По каждому рискованному пункту спроси: оставить / сузить / удалить.
Ничего не удаляй без согласия.
```

### Здоровье T-800

```text
/t800-doctor

Проверь установку T-800: версия, hooks, scripts, registry. Нужен ли Reload?
```

### Аудит плагина + фикс

```text
/t800-plugin-audit

Аудит плагина: /path/to/MyPlugin
Карта агентов/команд/skills, orphans, alwaysApply.
Отчёт в plugin-memory/audits/. Потом предложи fix-pack.

# после согласия:
/t800-fix
Примени последний fix-pack, PATCH only, без полного DEEP.
```

### Создать команду

```text
/t800-start

Создай /weekly-git-report для этого проекта (.cursor/commands/):
краткий отчёт по git log за 7 дней, readonly, с description «когда вызывать».
После сборки — validate и пример запуска.
```

### Создать skill

```text
/t800-start

Skill code-review-checklist в .cursor/skills/:
чеклист PR (security, tests, UX), без alwaysApply, с примерами.
```

### Создать субагента

```text
/t800-start

Readonly субагент docs-summarizer: читает docs/*.md, summary + риски, файлы не пишет.
```

### Сценарий на 30–40 минут

```text
/t800-bootstrap
/t800-doctor
/t800-audit
# решения keep/narrow/remove
/t800-fix
/t800-start   # создать нужную команду
```

Больше примеров: [README → Примеры промптов](../README.md#примеры-промптов).

## 7. Как создавать артефакты

1. Открыть нужный workspace (проект или checkout плагина).
2. `/t800-start` + задача обычным языком.
3. Уточнить surface, если спросят: плагин / проект / глобально.
4. Дождаться factory-auditor PASS.
5. При необходимости Reload (новые commands/agents).

Правка существующего — **`/t800-fix`**, не полный start.

## 8. Разбор Cursor и плагинов

| Цель | Команда | Куда отчёт |
|------|---------|------------|

## 11. Частые вопросы

**Нужен ли zip для обновления?**  
Нет. Авто при новом чате или `/t800-update`.

**Почему Reload?**  
Чтобы Cursor подхватил новые agents/hooks/commands.

**Можно править agents руками?**  
Лучше через `/t800-start` / `/t800-fix`, иначе память конвейера разъедется.

**Как отключить автообновление?**  
`T800_SKIP_AUTO_UPDATE=1`.
