# Обновление T-800

**С версии 1.15** вручную класть zip и «обновлять каждый раз» **не нужно**.

## Автомат (по умолчанию)

При **новом чате** hook сам сверяет GitHub и ставит новую версию.  
После этого: **Developer: Reload Window**.

Отключить: `export T800_SKIP_AUTO_UPDATE=1`

## Вручную (срочно)

```text
/t800-update
```

```bash
bash ~/.cursor/plugins/local/t-800-agent/scripts/t800-update-from-github.sh
```

## Офлайн / zip (запасной путь)

Если нет сети к GitHub — положите актуальный `t-800-agent-v*.zip` в папку и в Agent:

```text
Обнови T-800 из zip в этой папке: разархивируй, install-plugin.sh, verify-install.sh,
покажи version, попроси Reload Window.
```

Репозиторий: https://github.com/Khar-AG/t-800-agent  
Контракт: `shared/auto-update-contract.md`

## Release checklist (после правок KB)

Перед релизом / merge изменений в `knowledge-base/`:

```bash
cd t-800-agent
python3 scripts/t800_kb_provenance_gate.py --plugin-root .
# или офлайн fixtures:
python3 scripts/t800_kb_provenance_gate.py --fixture-dir tests/fixtures/kb-provenance/legal-sync
```

Gate: файл в diff `knowledge-base/` должен быть в `manifest.json` **или** иметь frontmatter `provenance: manual` + `author`. Exit 0 = OK.
